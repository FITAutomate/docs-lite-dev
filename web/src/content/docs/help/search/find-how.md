Use short, concrete terms first, then widen only if needed.

## Practical patterns

- Start specific: `invoice approval` instead of `process`
- Try alternate wording: `handover` and `transition`
- Use task words: `how to`, `checklist`, `publish`
- Search by page concept: `drafting`, `live edits`, `git push`

## Before creating a new page

Search first to avoid duplicates. If a page already exists, improve that page before creating a new one.

## If results are too broad

1. remove vague words
2. add a second keyword
3. open one likely page and follow internal links

Continue to [Writing](/writing/) when you are ready to draft or improve content.
