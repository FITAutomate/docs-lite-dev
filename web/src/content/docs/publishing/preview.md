Preview means checking that your update reads correctly before it goes live.

## What to verify

- headings are clear and in the right order
- lists and steps render correctly
- links go to the expected page
- wording matches the intended process

## Preview checklist

1. open the changed page
2. scan it as a first-time reader
3. test links you added or changed
4. confirm no accidental edits are included

When preview looks good, run [Git Status](/publishing/git-status/).
