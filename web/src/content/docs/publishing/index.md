Publishing in FIT Docs Lite follows a simple sequence:

1. preview your changes
2. check what changed
3. commit with a clear message
4. push updates to the remote repository
5. run or confirm your publish/deploy step

## Why this matters

This flow keeps updates understandable and safe. Everyone can see what changed, why it changed, and when it was published.

## Push and publish are different

- push updates Git history on the remote repository
- publish updates the live site
- your team may publish by direct branch deploy, pull request merge, or manual deploy trigger
