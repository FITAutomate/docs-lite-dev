A commit is a saved checkpoint with context.

For beginners, a helpful way to think about commit is: save your update with a message others can understand.

## Commit guidance

- keep commits focused on one topic
- use a short message that explains the change
- avoid combining unrelated edits

## Example

```bash
git commit -m "Improve getting started onboarding flow"
```

After commit, continue to [Git Push](/publishing/git-push/).
