Start here if you are new to this template.

This section helps you understand what FIT Docs Lite is, how content is organized, and how GitHub is used to make updates.

## First-run checklist

1. Read [What Is Here](/getting-started/what-is-here/) for a quick map of the library.
2. Read [What Is Included](/getting-started/what-is-included/) to understand what ships in the baseline.
3. Read [What Is Available](/getting-started/what-is-available/) to see what you can safely expand.
4. Use [Search](/help/search/) to find pages quickly.
5. Make a first small update using [Live Page Edits](/writing/live-page-edits/).
6. Follow [Publishing](/publishing/) to review and release your change.

## GitHub in plain language

In FIT Docs Lite, pages live in this repository as Markdown files. You can edit those files directly in GitHub.

When you update a page:

1. edit the file
2. review what changed
3. commit with a clear message
4. push the update
5. publish through your environment's deploy workflow (automatic or manual)

If you are ready to write, go to [Writing](/writing/).
