The baseline is intentionally small so teams can start fast and expand only when needed.

## Available now

- read and navigate a published documentation site
- edit pages directly in GitHub
- make focused updates to existing pages
- draft new content in Markdown
- review tracked changes before publishing
- publish updates through your configured deployment workflow

## What teams usually expand next

- add more pages under existing sections
- improve cross-links between related procedures
- add team-specific terms, examples, and standards

Keep expansions practical. Add content when it helps a reader complete real work.
