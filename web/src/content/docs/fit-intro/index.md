# FIT Docs Lite at a Glance

FIT Docs Lite is a practical documentation template for small teams that need clear pages and simple publishing through GitHub.

## Who this is for

- teams that have shared procedures but limited time
- people who can edit files in GitHub but are not developers
- teams that want tracked updates and predictable publishing

## What the template gives you

- a starter information architecture with `Getting Started`, `Writing`, and `Publishing`
- baseline pages you can keep, remove, or expand
- a beginner-safe workflow for making and shipping updates

## How the site and repo relate

- the published site is the reader view
- the Markdown files in this repository are the source
- when you edit a page in GitHub, you are editing the source for the site
- your changes are tracked, reviewed, committed, pushed, then deployed/published

## Recommended first-run path

1. Read [Getting Started](/getting-started/) for orientation.
2. Review [What Is Included](/getting-started/what-is-included/) to see your starting structure.
3. Make a small safe update using [Live Page Edits](/writing/live-page-edits/).
4. Publish with the steps in [Publishing](/publishing/).
