Live page edits are best for focused improvements, not full rewrites.

## Good live edit examples

- fix unclear wording
- correct an outdated step
- add one missing link
- tighten a heading for better scanning

## Safe edit workflow

1. open the existing page in GitHub
2. make a small change with clear intent
3. review the diff before saving
4. commit with a message that explains what changed
5. publish using the [Publishing](/publishing/) steps

## Keep risk low

- change one idea at a time
- avoid editing many pages in one commit
- preview before push when possible

If your update is larger, switch to [Drafting](/writing/drafting/).
